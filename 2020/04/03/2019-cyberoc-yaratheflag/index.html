[ cyberoc 2019 ] yaratheflag

Posted on 2020-04-03 Edited on 2020-04-05 In pwnable Disqus: 5.5k 5 mins.

[ 2019 cyberoc ] yaratheflag

Funny

↗binary summary

  • yara
  • tcache dup

↗Analyze

glibc 2.28, tcache환경이다.
일반적인 yara에 추가적으로 모듈을 포함시켜 빌드했는데,
모듈에서 취약점을 찾아 shell을 취득하면 된다.

↗Mitigation

got overwrite로 해야할 것 같다.

↗build

제공되는 Dockerfile과 Build_Yara.sh를 이용하면 간단하게 환경구성이 가능하다.
나는 좀더 편한 디버깅을 위해서 로컬을 사용했다.

↗patchfile

어떠한 패치를 진행하고 빌드를 한뒤 패치파일을 문제에 포함시켜줬다.

일부를 살펴보면 다음과 같이 secret.c라는 모듈을 추가했다.
해당 모듈을 분석하면 된다.

↗Memory block

secret.h파일을 보면 사용하는 구조체들을 확인할 수 있다.

블록 하나당 0x20만큼의 크기를 차지한다.
모든 block에 대해서 magic값에 대한 검사가 있다.

상위 4byte는 무조건 magic값이다.

↗Working

인자로 들어온 파일을 긁어와서 각각 구조체에 넣어준다.

block data를 긁어오고, 해당 블록을 토대로 Container_Parse를 진행한다.

↗Vuln

2가지 oob취약점이 존재한다.

↗Unit_Parse(0)

unit -> unitID를 조작하면 heap에서 oob를 만들 수 있다.

↗Container_Parse()

container -> containerID를 조작가능하다면, bss에서 oob를 트리거 할 수 있다.

↗UNIT struct trigger

취약점을 트리거하기 위해서는 Unit_Parse의 unit을 내가 원하는 block으로 지정해야한다.
module_load -> Container_Parse -> Container_readUnits -> Unit_Parse순으로 함수가 호출된다.

↗Container_readUnits()

container의 unitVector, unitCnt를 이용하면 내가 원하는 idx8개를 Unit으로 인자전달이 가능하다.

↗oob

중요한 취약점은 heap에서 일어나는 oob이다.
heap을 이용해 aaw를 만들어서 got overwrite를 하기 위해서는 tcache dup이 가장 좋은 방법이다.

heap영역 아무곳에 원하는 값을 넣을 수 있는 상황이면,
tcache entry chunk의 0x20짜리 bin에 넣으면 malloc(0x10)을 통해 aaw을 만들 수 있다.

↗makeStr()

내용을 buffer에 복사하기 위해서 자체적으로 makeStr이라는 함수를 사용한다.

여기서 malloc(0x10)을 할 수 있는데, oob를 0x20짜리 bin에 넣은이유가 이때문이다.

다른 size로는 할당불가, 0x20 고정

↗Got Overwrite

aaw가 가능하므로 yr_finaliz@got를 주어진 원샷으로 덮으면 쉘취득이 가능하다.

↗SetAllVars()

마무리 루틴으로 SetAllVars를 호출하는데,
containers[0x100] 배열에 값이 존재하면 별도의 처리과정을 해준다.
이때 값이 존재한다면 error scanning test: error: 4구문과 함께 오류를 띄어준다.
main에서 yr_finalize를 호출하기때문에 쉘취득과는 상관이없지만,
익스를 짤때 꼬여버릴 수 있기때문에 containers[]의 인덱스를 containers[0x100]로 맞추어서 SetAllVars루틴을 넘어가도록했다.

↗Exploit

분석 진짜 재밌다… 라업 없이 풀어보길…
