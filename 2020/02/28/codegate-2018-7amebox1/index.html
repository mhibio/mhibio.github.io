[ codegate 2018 ] _7amebox1

[ Codegate 2018 ] _7amebox1

disgusting....

↗binary summary

  • VM prob

↗Analyze

python으로 에뮬레이터를 만들어뒀는데
구성환경과 제공된 OPCODE를 분석하고, 분석된 내용을 바탕으로 shell을 취득하면 된다.

↗binary

  • _7amebox.py - vm 환경 구성
  • flag - flag
  • mic_check.firm - vm 환경의 펌웨어 (opcode)
  • vm_name.py - 실행 프로그램

↗class EMU:

  • Nx bit 보호기법
  • r0 ~ r10, bp, sp, eflags, pc, zero 총 16개의 레지스터
  • 0x100000 size의 Memory관리
  • File system 존재
  • 6개의 syscall
  • 31개의 instruction
  • 모든 연산에 0b1(*21)과 and연산을 해주는걸로 7bit기반 vm 이라는것을 유추

↗EMU.syscall_table:

  • exit
  • open
  • write
  • read
  • allocate
  • random

↗EMU.func:

  • pipe, mitigation, permission, loader, file stream 관련 함수
  • execute - opcode 따라 차례대로 실행
  • dispatch - 해당 opcode를 unpack
    • TYPE
    • op
    • size
    • oper[2]
  • op_x1 ~ op_x30, sys_s0 ~ sys_s5 - 연산, syscall

↗disassembler

vm_name.py는 mic_check.firm을 읽고, 해당 firmware를 토대로 에뮬레이터가 돌아간다.
mic_check.firm이 무슨 연산을 하는지 확인하기 위해서 disassembler를 짤 필요가 있다.
=> op_x1 ~ op_x30 을 토대로 디스어셈블러 제작

이거 이해하면서 짜는데 2일이 걸렸다.

↗mic_check.firm

↗vuln

  • overflow
  • canary
  • enable Nx

입력받는 루틴이 한번 있는데
=> 입력은 bp - 0x3c 에서 받지만, size가 0x42이므로 overflow존재

bp - 0x3 - sp + 0x39위치에 카나리가 존재함
=> 0x12345로 고정이므로 우회가능

Nx mitigation이 False로 설정되어있음 + 별도의 ASLR설정 X
=> opcode넣고 lob 1번 풀듯이 ORW해주면 된다.
