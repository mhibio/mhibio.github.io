¶Analyze

¶Main

입력을 받고, 입력값이랑 여러 전역변수들을 인자로 VM을 구현한 함수를 호출한다.

¶Vm

다음과 같이 VM환경이 구성되어있는데, 입력값을 토대로 opcode에 따라서 14개의 instruction이 존재한다.
지역변수로 custom_stack을 구성해서 작동하며, 전역변수와 지역변수에서 csp, cbp등을 관리한다. (custom sp, custom bp)
함수를 하나하나 열어보면 코드가 직관적이여서 쉽게 분석이 가능하다.

¶Vuln

stack상태는 다음과 같다.

NULL로 채워진 0x50 bytes공간이 custom stack으로 쓰인다.
stack은 높은 주소에서 낮은주소로 쓰이므로
일반적으로 함수의 ret을 건드릴 방법은 없어보인다.

¶op: 0x28

0x28 opcode는 다음과 같은작업을 한다.

csp를 0x50 만큼 증가시키는데, 2번 실행시킨다면 기존의 csp는 0x7fffffffe240에서 0x7fffffffe2e0를 바뀔 것 이다.

stack을 ret보다 높은주소로 옮기고, 다른 instruction으로 ret을 덮을 수 있다.

¶Exploit Code

나머지 instruction이랑 main ret (__libc_start_main+240)을 잘 활용해서 원샷을 만들고
canary안건드리게 해서 함수 return에다가 박으면 끝

VM문제들도 이제 익숙해져간다.
