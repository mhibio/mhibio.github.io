¶Nohack

쉽게 생각해서_libc_IO_vtables의 권한을 Read Permission만 준다는 이야기이다.

¶3줄 요약

stdout의 vtable filed를 write권한이 남아있는 _IO_helper_jumps로 돌리고, ( vtable bypass )
_IO_helper_jumps의 함수들은 setcontext+53으로 overwrite
그후 rop와 orw

¶Input

Input을 총 4번 줄 수 있음

¶aaw1
  • _IO_buf_base를 stdin으로 조작 - (_IO_buf_end변경을 위해서)

이제부터 input은 stdin으로 들어감

¶aaw2 - 1
  • _IO_buf_base를 stdout으로, _IO_buf_end를 충분한 크기로 조작 ( stdin + 0x2000 )

이제부터 input은 stdout으로 들어감

¶aaw2 - 2
  • vtable덮고 나머지 payload적어줌

fclose -> _IO_new_fclose -> _IO_new_file_close_it -> call QWORD PTR [ vtable+0x88 ] -> _IO_helper_jumps+0x88 -> rop - mprotect -> orw

¶_IO_new_file_close_it

이때 rbp는 stdout의 vtable field값을 가진다.

¶setcontext+53

orw를 수행하기위해서는 실행가능한 메모리 영역이 필요
-> mprotect함수를 이용해서 rwx부여가 필요
-> setcontext+53은 이를 위한 아주 좋은 가젯 XD

rsp(rdx+0xa0)하나만 조작가능하면 맘대로 rop수행 가능

  • _IO_new_file_close_it함수에서 rdx는 stdout의 vtable field값을 가진다.
